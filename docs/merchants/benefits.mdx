# Why use Stitch Payments?

Stitch Payments for WooCommerce brings enterprise-grade card processing to your WooCommerce store without custom theme code.

## Secure checkout

- **Hosted card fields** — The card number is entered in a CardPointe tokenizer iframe. Full card numbers and CVV are not stored on your WordPress site.
- **Tokenized payments** — Each charge uses a secure token from CardPointe, reducing PCI scope for your store.
- **Transparent data handling** — Order cleanup removes sensitive checkout fields after every payment attempt; see [Data and privacy](/docs/merchants/data-and-privacy).
- **Test mode** — Validate your integration in a UAT environment before going live.

## Better customer experience

- Clean, mobile-friendly payment form on classic and Blocks checkout
- Optional animated **card preview** while customers enter details
- **Saved cards** for logged-in customers and faster repeat purchases
- Automatic field advance when card number, expiry, and CVV are valid
- Accepted card brand icons (Visa, Mastercard, American Express, Discover)

## Merchant control

Configure branding and behavior from **WooCommerce → Settings → Payments → Stitch**:

- Payment title and description
- Logo, card icons, and in-form description toggles
- Custom logo URL and field label overrides
- Stored payment methods and surcharge disclosure
- Live **checkout preview** in admin — see changes before you save

## Operations and reporting

- Transaction references on orders for support and reconciliation
- Refund and void handling from the WooCommerce order screen
- Webhook URL for CardPointe transaction notifications
- WooCommerce Subscriptions renewal support
- HPOS (custom order tables) compatible

## Built for WooCommerce

Unlike theme-embedded solutions, this plugin is a **standalone extension**. It works across themes, supports WooCommerce Blocks, and can be updated independently of your theme.
