# Changelog

All notable changes to **Stitch Payments for WooCommerce** are documented here.

## 0.3.0 — 2026-06-25

Internal architecture, developer tooling, and surcharge hardening. No change to checkout behavior.

### Architecture

- Refactored the monolithic gateway into focused service classes: `Admin_Preview`, `Settings_Fields`, `Checkout_Display`, `Payment_Processor`, `Account_Payment_Methods`, `Subscription_Support`, `Surcharge_Service`, and `Order_Meta_Box`
- The `Gateway` class shrank from roughly 2,630 to about 1,000 lines, with each responsibility now isolated and independently testable

### Surcharge

- The disclosed credit card surcharge is now derived from the **server-side cart total** instead of trusting the posted amount
- AJAX surcharge responses now use standard `wp_send_json_success()` / `wp_send_json_error()` envelopes
- Fixed a unit mismatch that could overstate the surcharge rate, and a case where a non-eligible card could still show a fee

### Developer API

- New filter `woocommerce_stitch_register_gateway` to enable or disable gateway registration (useful for staging or conditional rollouts), replacing a hardcoded flag
- **Changed:** checkout-presentation helpers (`get_payment_form_i18n()`, `get_saved_payment_methods_for_checkout()`, `get_checkout_display_settings()`, `get_iframe_css()`, and related) moved from `Gateway` to the new `Checkout_Display` class, accessible via `$gateway->checkout_display()`

### Quality & tooling

- Added PHP_CodeSniffer (WordPress + WooCommerce), PHPStan, and a PHPUnit unit-test suite (all excluded from the release package)
- Sanitized admin settings-screen detection query parameters

### Documentation

- Removed legacy child-theme references from code and docs
- Clarified data-retention and activation / deactivation / uninstall behavior
- Bumped tested-up-to to WordPress 6.9

## 0.2.0 — 2026-06-10

### Admin

- Redesigned gateway settings page with branded hero, status pills, and tabbed **Test** / **Production** credentials
- Live **Checkout preview** panel with AJAX updates before save; sticky sidebar on desktop
- Reorganized settings: **Checkout display** (including card preview), **Other settings** (stored payments, surcharge, debug logging)

### Checkout display

- Toggle Stitch logo, accepted card icons, in-form description, and card preview
- Custom checkout logo URL and field label overrides (card number, expiry, CVV, cardholder name)
- Classic template via `wc_get_template` with theme override at `woocommerce/stitch/payment-form.php`
- Shared display settings and i18n for classic and Blocks checkout

### Developer API

- Filters: `woocommerce_stitch_payment_form_template_args`, `woocommerce_stitch_saved_payment_methods`, `woocommerce_stitch_payment_form_i18n`, `woocommerce_stitch_accepted_card_brands`, `woocommerce_stitch_payment_header_html`, `woocommerce_stitch_is_checkout_context`, and related checkout/asset filters
- Actions: `woocommerce_stitch_before_payment_form`, `woocommerce_stitch_after_payment_header`, `woocommerce_stitch_before_new_card_fields`, `woocommerce_stitch_after_payment_form`
- `Checkout_Context` class for WooCommerce checkout, add-payment-method, shortcode, and Elementor widget detection
- Public gateway helpers (`get_payment_form_i18n()`, `get_saved_payment_methods_for_checkout()`, etc.)
- `Card_Icons::kses_payment_header_html()` for SVG-safe payment header output
- In-plugin guide at `docs/DEVELOPERS.md`

### Checkout UX

- Auto-advance focus from card number → expiry → CVV → cardholder name when each field passes validation

### Packaging & i18n

- `package.php` release zip builder with `.distignore` for Windows-friendly packaging
- Fixed `translators:` comments for surcharge disclosure strings

### Fixed

- Accepted card icon SVGs were stripped from the payment header by `wp_kses_post()` — header output now uses a dedicated allowlist

### Documentation

- Merchant and developer documentation site updated to match 0.2.0 (this site)

## 0.1.0 — 2026-06-05

Initial standalone plugin scaffold.

### Added

- WooCommerce payment gateway (`stitch`) with test and production credentials
- WooCommerce dependency and activation checks
- HPOS (custom order tables) compatibility declaration
- WooCommerce Cart & Checkout Blocks compatibility
- CardPointe API client with sanitized logging paths
- Custom payment token type `WC_Payment_Token_Stitch` for saved cards
- Classic checkout payment form with hosted card tokenizer
- WooCommerce Blocks payment method integration
- Payment processing and saved-token creation
- Refund and void handling with admin order transaction details
- WooCommerce Subscriptions renewal hooks and scheduled renewal processing
- Surcharge calculation, session disclosure, and order-meta persistence
- Data retention settings and uninstall policy options

### Notes

This version established the migrated gateway for local and staging verification. Complete manual testing for classic checkout, Blocks checkout, saved cards, refunds, voids, subscriptions, surcharge disclosure, HPOS, and failure paths before production use.
